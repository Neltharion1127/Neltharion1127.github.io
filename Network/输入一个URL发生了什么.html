# 输入一个URL发生了什么

# 1.有缓存

直接从缓存中取出js、html、css文件展示出来

# 2.浏览器查找域名的IP地址

  1. 解析域名，首先查看host文件中是否有对应规则，如果有就是用host文件里面的ip地址
  2. 如果host文件中没有ip地址，浏览器会发出一个DNS请求DNS服务器
  3. 本地DNS服务器会首先查询它的缓存记录中是否存在记录，如果存在就可以直接返回结果，如果没有，本地DNS服务器还要向DNS根服务器进行查询
  4. 根DNS服务器没有记录具体的域名和IP的对应关系，而是告诉本地DNS服务器，你可以到域服务器上继续查询，并给出域服务器的地址。这种过程是迭代的过程。
  5. 本地DNS服务器继续向服务器发出请求，例如.com域名，请求的对象就是.com域服务器。.com域服务器收到请求之后，也不会直接返回域名和IP的对应关系，而是告诉根底DNS服务器，你的域名的解析服务器的地址。
  6. 最后，本地服务器向域名的解析服务器发出请求，这是就能收到一个域名和IP地址对应关系，本地DNS服务器不仅要把IP地址返回给用户电脑，还要把这个对应关系保存在缓存中，以备下次别的用户查询时，加快网络访问

# 3.浏览器向web服务器发送一个HTTP请求

拿到域名对应的ip地址之后，浏览器会以一个随机端口（1024-65535）向服务器的Web程序（如nginx）80端口发起TCP连接请求。当这个请求到达服务端后（这中间通过各种路由设备，局域网除外），进入到网卡，然后试进入到TCP/IP协议栈（一层一层的解包），最终到达web程序，建立TCP/IP的连接。
